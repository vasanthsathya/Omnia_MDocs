# Operations & Maintenance

Day-2 operations for managing a running Omnia cluster. These guides cover common administrative tasks you will perform after the initial deployment is complete -- scaling the cluster, re-provisioning nodes, managing logs, hardening security, and cleaning up the OIM when a fresh start is needed.

!!! tip
    If you have not yet deployed your cluster, start with the [Get Started](../GetStarted/index.md) tutorials. The procedures in this section assume a working Omnia environment.

## Topics

- [Add/Remove Nodes](add_remove_nodes.md) - Scale your cluster by adding or removing compute nodes
- [Best Practices Checklist](best_practices_checklist.md) - Operational best practices and recommendations
- [Log Management](log_management.md) - Collect and manage system logs
- [OIM Cleanup](oim_cleanup.md) - Clean up the OIM for a fresh deployment
- [Reprovision Cluster](reprovision_cluster.md) - Re-provision the entire cluster
- [Security Hardening](security_hardening.md) - Security hardening procedures
